# Руководство по стилю LaTeX в данном документе

## Общие требования

- Каждый `.tex`-файл должен начинаться с комментария, указывающего корневой файл:
  ```latex
  % !TeX root = ../main.tex
  ```
  или
  ```latex
  % !TeX root = ../../main.tex
  ```

## Параметры вёрстки и требования ГОСТ

- Все правила и примеры должны соответствовать фактической конфигурации проекта, заданной в `main.tex` и `preamble/*.tex`.
- Документ собирается как `extarticle` c кеглем `14pt`.
- Поля страницы фиксированы:
  - левое — `2.5 cm`;
  - правое — `2.5 cm`;
  - верхнее — `2.5 cm`;
  - нижнее — `2.5 cm`.
- Межстрочный интервал — полуторный.
- Основной текст и математика настроены под Times-подобную гарнитуру; локальные переопределения шрифтов без необходимости не допускаются.
- Абзацный отступ должен соответствовать требованиям ГОСТ; в проекте целевое значение — `1.25 cm`.
- Подписи, списки, библиография, перекрёстные ссылки и сноски должны оформляться через существующие настройки проекта, а не вручную поверх них.

## Типографика

### Кавычки

- Использовать `\enquote{}` для кавычек:
  ```latex
  \enquote{информатика}
  ```

### Тире и дефисы

- Дефис: `-` (в словах: `что-то`)
- Тире: `---` (в предложениях, с неразрывным пробелом)
- Диапазоны: `--` (между числами: `10$--$20`)

### Многоточие

- Использовать `\dots` или `\ldots` вместо `...`

## Числа и единицы измерения

- Дробные числа не следует писать как `1,2`, так как LaTeX интерпретирует запятую в математическом режиме не как десятичный разделитель. Вместо этого следует использовать пакет `siunitx`:
  - `\num{1.2}` — просто число;
  - `\qty{1.2}{\meter}` — число с единицей измерения (кг, МБ, Дж и т.п.).
  - **Важно:** `siunitx` создаёт неразрывный пробел при разделении тысяч, например `1 000,24`.
    Если такое поведение не подходит, например в других системах счисления, дробные числа следует записывать так:
    - `3423{,}222` — просто число
    - `1000~0010~1000{,}001` — пример дробного числа в двоичной системе счисления

- Для диапазонов чисел использовать `\numrange{}{}`:
  ```latex
  \numrange{1}{10}       % выведет: 1–10
  \numrange{0.6}{1.3}    % выведет: 0,6–1,3 (запятая автоматически)
  ```

- Для диапазонов с единицами измерения использовать `\qtyrange{}{}{}`:
  ```latex
  \qtyrange{10}{20}{\percent}  % выведет: 10–20 %
  \qtyrange{1}{5}{\MB}         % выведет: 1–5 МБ
  ```

- Для степеней двойки использовать математическую запись: `$2^{10}$`, а не `2^10` в тексте.

- Примеры единиц для информатики (определены в `preamble/siunitx.tex`):
  ```latex
  \qty{8}{\bit}      % биты
  \qty{1}{\byte}     % байты
  \qty{100}{\bps}    % бит/с
  \qty{100}{\Bps}    % байт/с
  \qty{1}{\KB}       % килобайты
  \qty{1}{\MB}       % мегабайты
  \qty{1}{\GB}       % гигабайты
  ```

## Формулы

### Одиночные формулы

- Использовать `\[ ... \]` для формул в отдельной строке:
  ```latex
  \[
      I = - \sum^{N}_{i=1}p_{i}\times \log_{2}{p_{i}}
  \]
  ```

- Для нумерованных формул использовать окружение `equation`:
  ```latex
  \begin{equation}
      E = mc^2
  \end{equation}
  ```

### Формулы с пояснением «где»

Следует использовать готовое окружение `where`:

```latex
\[
    I = - \sum^{N}_{i=1}p_{i}\times \log_{2}{p_{i}},
\]
\begin{where}
    \item [$I$] --- количество информации, выраженное в битах (в $\log_{k}p_{i}$ $k = 2$);
    \item [$N$] --- число состояний системы;
    \item [$p_{i}$] --- вероятность (относительная частота) перехода системы в $i$-е состояние.
\end{where}
```

### Математические обозначения

- Векторы следует оформлять через `\mathbf{}` (в проекте переопределено `\let\vec=\mathbf`).
- Греческие буквы: `\alpha`, `\beta`, `\gamma`, `\delta`, `\pi`, `\sigma` и т.д.
- Логические операции:
  - Конъюнкция: `\wedge` или `\land`
  - Дизъюнкция: `\vee` или `\lor`
  - Отрицание: `\neg` или `\overline{x}`
  - Исключающее ИЛИ: `\oplus`
  - Импликация: `\to` или `\rightarrow`
  - Эквивалентность: `\equiv` или `\leftrightarrow`

## Списки и перечни

### Маркированные списки

- Использовать `itemize` для маркированных списков:
  ```latex
  \begin{itemize}
      \item Первый пункт;
      \item Второй пункт.
  \end{itemize}
  ```

### Нумерованные списки

- Использовать `enumerate` для нумерованных списков:
  ```latex
  \begin{enumerate}
      \item Первый пункт;
      \item Второй пункт.
  \end{enumerate}
  ```

- В этом проекте нумерованные списки настроены следующим образом:
  - 1-й уровень: `а)`, `б)`, `в)` ...;
  - 2-й уровень: `а)`, `б)`, `в)` ...;
  - 3-й уровень: `1)`, `2)`, `3)` ....

### Списки определений

- Использовать `description` для терминов и определений:
  ```latex
  \begin{description}
      \item [Информатика]
            Дисциплина, изучающая свойства и структуру информации.
      \item [Алфавит]
            Конечное множество различных знаков.
  \end{description}
  ```

- В маркированных списках используется маркер `--`.
- У списков по умолчанию убраны лишние вертикальные интервалы; не добавлять их вручную без необходимости.
- Для терминов и определений следует использовать `description`, так как это окружение уже настроено для единообразного оформления.

## Таблицы

### Базовые таблицы

- Использовать `tabular` для простых таблиц:
  ```latex
  \begin{table}[H]
      \centering
      \begin{tabular}{|c|c|c|}
          \hline
          A & B & C \\
          \hline
          1 & 2 & 3 \\
          \hline
      \end{tabular}
  \end{table}
  ```

### Многостраничные таблицы

- Использовать `xltabular` для таблиц, занимающих несколько страниц:
  ```latex
  \begin{xltabular}{\textwidth}{|c|c|C|}
      \caption{Название таблицы}\label{tab:label} \\
      \hline
      Заголовок 1 & Заголовок 2 & Заголовок 3 \\
      \hline
      \endfirsthead
      %
      \multicolumn{3}{r}{\small Продолжение таблицы~\thetable} \\
      \hline
      Заголовок 1 & Заголовок 2 & Заголовок 3 \\
      \hline
      \endhead
      %
      Данные & Данные & Данные \\
      \hline
  \end{xltabular}
  ```

### Оформление таблиц

- Использовать `\hline` для горизонтальных линий.
- Для вертикального центрирования текста в ячейках использовать `m{}` или `>{\centering\arraybackslash}X`.
- Для жирного заголовка использовать `\textbf{}`.
- Для переноса слов в ячейках использовать `p{ширина}` или `X`-колонки.

- Предопределённые типы колонок (в `main.tex`):
  ```latex
  \newcolumntype{C}{>{\centering\arraybackslash}X}  % по центру
  \newcolumntype{L}{>{\raggedright\arraybackslash}X} % по левому краю
  \newcolumntype{R}{>{\raggedleft\arraybackslash}X}  % по правому краю
  ```

## Рисунки

### Вставка изображений

- Использовать окружение `figure`:
  ```latex
  \begin{figure}[H]
      \centering
      \includegraphics[width=0.5\textwidth]{image-name}
      \caption{Название рисунка}
      \label{fig:label}
  \end{figure}
  ```

### Обтекание текстом

- Использовать `wrapfigure` для обтекания рисунком:
  ```latex
  \begin{wrapfigure}{l}{0.25\textwidth}
      \centering
      \vspace{-\intextsep}
      \includegraphics[width=0.25\textwidth]{image-name}
      \caption*{Подпись}
  \end{wrapfigure}
  ```

### Подписи

- Подписи к рисункам располагаются **снизу**.
- Подписи к таблицам располагаются **сверху**.
- Между номером и текстом подписи используется тире.
- Текст подписи оформляется курсивом.
- Подписи к таблицам выравниваются по ширине, к рисункам — по центру.

## Листинги кода

- Использовать `minted` для вставки кода (требует `-shell-escape`):
  ```latex
  \begin{minted}{python}
  def hello():
      print("Hello, World!")
  \end{minted}
  ```

- Для inline-кода использовать `\mintinline` или `\verb`:
  ```latex
  \mintinline{python}{print()}
  \verb|print()|
  ```

- Курсив в листингах отключен (настроено в `main.tex`).

- **Важно:** `\verb` и `\mintinline` нельзя использовать в аргументах команд и окружений, например в `\caption`, `\section` и `\footnote`.

- **Альтернативы для аргументов:**
  - Использовать `\texttt{}` для моноширинного текста:
    ```latex
    \section{Работа с \texttt{printf}}
    \caption{Функция \texttt{main}}
    ```
  - Экранировать специальные символы: `\{`, `\}`, `\#`, `\$, `\%`, `\&`, `\_`, `\textasciitilde{}`, `\textasciicircum{}`

## Ссылки и перекрёстные ссылки

- Использовать `\label{}` для меток.
- Использовать `\ref{}` или `\cref{}` для ссылок; `\cref` автоматически добавляет тип объекта.
- Метки именовать осмысленно:
  - `\label{sec:introduction}` — для секций;
  - `\label{subsec:background}` — для подсекций;
  - `\label{fig:architecture}` — для рисунков;
  - `\label{tab:comparison}` — для таблиц;
  - `\label{eq:shannon}` — для формул;
  - `\label{lst:example}` — для листингов.

### Разница между `\ref` и `\cref`

| Команда | Результат | Когда использовать |
|---------|-----------|-------------------|
| `\ref{fig:graph}` | `1` | Когда тип объекта ясен из контекста |
| `\cref{fig:graph}` | `рис. 1` | Когда нужно явно указать тип объекта |
| `\ref{tab:data}` | `2` | В подписях, заголовках |
| `\cref{tab:data}` | `табл. 2` | В основном тексте для ясности |
| `\ref{sec:theory}` | `3` | При ссылке внутри той же секции |
| `\cref{sec:theory}` | `§3` | При ссылке на удалённую секцию |
| `\ref{eq:formula}` | `5` | Когда нужны только номер или номер в собственном оформлении |
| `\cref{eq:formula}` | `(5)` | В тексте с пояснением |

**Примеры:**
```latex
% В тексте — лучше cref для ясности:
Как показано в \cref{fig:graph}, зависимость линейная.
В \cref{tab:data} представлены результаты.

% В подписях и заголовках — ref (тип уже ясен):
\caption{Зависимость из \cref{fig:graph}} % или просто \ref{fig:graph}
\section{Сравнение с \cref{sec:theory}}

% В формулах — ref (чтобы не загромождать):
По формуле~\ref{eq:shannon} вычисляем...
```

**Рекомендация:** в основном тексте использовать `\cref`, а в подписях и формулах — `\ref`, если требуется более компактная запись.
- Фактические форматы ссылок в проекте:
  - разделы и подразделы — `§...`;
  - рисунки — `рис.~...`;
  - таблицы — `табл.~...`;
  - формулы — `(...)`;
  - приложения — `прил.~...`.
- Использовать неразрывный пробел перед `\ref`, `\cref` и `\cite`, когда ссылка идёт после слова: `в~\cref{...}`, `по формуле~\ref{...}`, `в источнике~\cite{...}`.

## Библиография

- Использовать `biblatex` с `biber`.
- Для оформления библиографии использовать уже настроенный стиль `gost-numeric`.
- Записи в `.bib`-файле должны содержать обязательные поля:
  ```bibtex
  @book{Cormen2013,
      author    = "Томас Х Кормен and Чарльз Эрик Лейзерсон and Рональд Линн Ривест",
      title     = "Алгоритмы: построение и анализ",
      edition   = "3",
      address   = "Москва",
      publisher = "ООО ''И. Д. Вильямс''",
      year      = "2013",
      pagetotal = "1328",
      language  = "russian",
  }

  @online{Top500,
      title    = "Top 500 supercomputers list",
      url      = "https://www.top500.org/",
      urldate  = "2023-02-12",  % ← дата обращения в формате ГГГГ-ММ-ДД
      language = "english",
  }
  ```

- Для онлайн-источников следует использовать поле `urldate` вместо `addendum` — дата обращения добавляется автоматически в формате «Дата обращения: 12.02.2023».
- Для онлайн-источников использовать тип `@online`; в библиографии такие записи оформляются с пометой `[Электронный ресурс]`.
- Ссылки на источники: `\cite{Cormen2013}` или `\cite[с.~123]{Cormen2013}`.
- Не следует добавлять `doi`, `isbn` и аналогичные поля только ради полноты, если они не нужны по содержанию: текущая конфигурация их не выводит.
- Язык источника указывать явно через `language`, чтобы сортировка и оформление оставались предсказуемыми.
- Русскоязычные и иноязычные источники сортируются по-разному; ручное форматирование записей не должно нарушать эту логику.

## Секции и заголовки

- Использовать стандартную иерархию:
  ```latex
  \section{Название}
  \subsection{Название}
  \subsubsection{Название}
  \paragraph{Название}
  ```

- Метки следует ставить **на той же строке**, что и заголовок секции:
  ```latex
  % Правильно:
  \section{Введение}\label{sec:introduction}
  \subsection{Предпосылки}\label{subsec:background}

  % Допустимо, но менее предпочтительно:
  \section{Введение}
  \label{sec:introduction}
  ```

- **Важно:** `\label` должен идти сразу после команды секции, без дополнительного текста между ними.

## Задания, примеры и решения

### Окружение для развёрнутых примеров

Использовать окружение `example` с автоматической нумерацией в рамках секции:

```latex
\begin{example}
  \task%
  Текст задания.

  \solution%
  Текст решения.

  \answer{} Текст ответа.
\end{example}
```

Команды внутри окружения:
- `\task` — выводит **Задание:** курсивом и подчёркнутым, текст следует за меткой
- `\solution` — выводит **Решение:** курсивом и подчёркнутым, текст следует за меткой
- `\answer` — выводит **Ответ:** курсивом и подчёркнутым, текст следует за меткой

> **Примечание:** Если текст задания, решения или ответа начинается с новой строки, завершайте команду комментарием `%`, чтобы избежать лишнего пробела: `\task%`.

Нумерация примеров автоматическая, сбрасывается после каждой `\subsubsection`.

### Inline-команда для примеров в тексте

Для кратких примеров внутри текста используется команда `\ex`:

```latex
Текст с примером: \ex%
$53 = 21 + 41$

\ex%
$+5_{10} = 0101_{2}$
```

Команда `\ex` выводит **Пример N:** с автоматической нумерацией.

> **Примечание:** После `\ex` добавляйте `%` для избежания предупреждений chktex.
