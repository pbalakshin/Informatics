# CONTRIBUTING

Спасибо за вклад в репозиторий `Informatics`.

Этот проект содержит учебно-методические материалы по дисциплине «Информатика», включая учебное пособие в LaTeX, лабораторные работы и лекционные материалы. Основная цель изменений: повышать качество материалов без нарушения структуры курса, сборки и оформления.

## Что можно улучшать

- исправления опечаток, фактических ошибок и неточностей;
- улучшение формулировок и структуры текста;
- дополнение иллюстраций, схем и примеров;
- исправления LaTeX-разметки и ссылок;
- улучшение CI/CD и сопутствующей инфраструктуры репозитория.

## Перед началом работы

1. Проверьте, нет ли уже открытого issue или pull request по той же теме.
2. Для крупных изменений сначала опишите предложение в issue.
3. Делайте изменения минимальными и тематически цельными.

## Локальная проверка

Основной документ собирается из директории `Book/`.

```bash
cd Book
latexmk -pdf main.tex
```

Для очистки временных файлов:

```bash
latexmk -pdf -c main.tex
```

Если в тексте используются подсветка кода и связанные инструменты, убедитесь, что установлены совместимые версии TeX Live и Pygments, как указано в `README.md`.

## Рекомендации по оформлению изменений

- придерживайтесь существующей структуры каталогов и именования файлов;
- не смешивайте содержательные правки и крупный рефакторинг в одном pull request;
- сохраняйте стиль и терминологию существующих материалов;
- не удаляйте исторические материалы из `_OLD_Informatics`, если это не согласовано отдельно;
- при изменении `Book/` проверяйте, что PDF по-прежнему собирается без ошибок и предупреждений о битых ссылках.

При изменении материалов в `Book/` необходимо соблюдать требования из `Book/STYLEGUIDE.md`. Это относится к структуре LaTeX-кода, типографике, ссылкам, библиографии и правилам оформления, связанным с ГОСТ.

## Commits and pull request

Предпочтительный порядок работы:

1. Создайте отдельную ветку от `main` или `master`.
2. Внесите изменения по одной теме.
3. Проверьте локальную сборку.
4. Откройте pull request с кратким описанием:
   - что изменено;
   - зачем это нужно;
   - как это проверялось.

Если изменение влияет на итоговый PDF, приложите скриншоты страниц или кратко опишите ожидаемый визуальный результат.

## Что важно для ревью

- корректность фактов и терминологии;
- отсутствие сломанных ссылок, label/citation warnings и ошибок сборки;
- соответствие материалов учебному назначению репозитория;
- аккуратность и воспроизводимость изменений.
